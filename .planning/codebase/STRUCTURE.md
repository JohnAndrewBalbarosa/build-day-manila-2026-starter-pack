## Directory Purposes

**`core/`:**
- Purpose: Reusable frame capture abstraction for practice and live modes
- Contains: Python package with ffmpeg subprocess wrapper (practice) and LiveKit client (live)
- Key files: `core/src/core/frame.py` (Frame dataclass), `core/src/core/practice.py` (local camera), `core/src/core/stream.py` (remote video)
- Status: Read-only reference implementation; do not edit

**`api/`:**
- Purpose: Typed HTTP SDK for communication with the game server
- Contains: Async httpx client, Pydantic validation models, domain-specific exceptions
- Key files: `api/src/api/client.py` (CasperAPI class), `api/src/api/models.py` (Feed/GuessResult/exceptions)
- Status: Read-only reference implementation; do not edit

**`agent/`:**
- Purpose: Your customizable AI agent that analyzes frames and submits guesses
- Contains: CLI orchestrator and your prompt/analysis logic
- Key files: `agent/src/agent/__main__.py` (entry point), `agent/src/agent/prompt.py` (your code here)
- Status: Editable; intended for user customization

**`.planning/codebase/`:**
- Purpose: Architecture analysis and documentation directory
- Contains: ARCHITECTURE.md, STRUCTURE.md, and future quality/concerns analyses
- Status: Generated by GSD analysis tools; read-only

## Naming Conventions

**Files:**
- `__init__.py`: Package entry point, lists public exports
- `__main__.py`: Executable entry point (invoked via `python -m package`)
- `*_test.py` or `test_*.py`: Test files (pattern not yet in use, follow this if adding tests)
- Module files: Descriptive lowercase with underscores (frame.py, practice.py, stream.py, client.py, models.py, prompt.py)

**Directories:**
- `src/[package-name]/`: Source code per PEP 517 structure (src layout)
- Workspace root contains 3 packages: `core`, `api`, `agent`
- Private implementation details: No special prefix convention; export only what's needed in __init__.py

**Functions:**
- Public API: snake_case, no leading underscore (start_practice, start_stream, analyze, guess)
- Private helpers: Leading underscore (\_detect_ffmpeg, \_build_capture_cmd, \_pump_video_to_queue)
- Async functions: Same naming as sync equivalents (async def start_practice, async def analyze)

**Variables:**
- Module-level constants: UPPERCASE (SYSTEM_PROMPT, \_MIN_FRAME_INTERVAL_S, \_FIRST_FRAME_TIMEOUT_S)
- Local variables: snake_case (frame_queue, pump_tasks, response, etc.)

**Types:**
- Dataclasses: PascalCase (Frame)
- Exception classes: PascalCase (Unauthorized, NoActiveRound, MaxGuessesReached, JudgeUnavailable)
- Pydantic models: PascalCase (Feed, GuessResult)

**Class Methods:**
- Public methods: snake_case (from_env, get_feed, guess, close)
- Dunder methods: Python convention (__init__, __str__)

## Where to Add New Code

**New Feature (requires server integration):**
- Primary code: `agent/src/agent/__main__.py` (add new mode or CLI flag)
- Configuration: `.env` file (add new env vars as needed)
- Example: Adding a "analyze and log confidence" feature would add logic to analyze() in `agent/src/agent/prompt.py`

**New Component/Module (within agent package):**
- Implementation: `agent/src/agent/new_module.py`
- Export in: `agent/src/agent/__init__.py`
- Example: If building a "context accumulator" to track recent guesses, create `agent/src/agent/context.py` and import in __main__.py

**Vision LLM Integration:**
- Primary location: `agent/src/agent/prompt.py`
- Customization: Implement the analyze() function with your LLM client
- Example: Initialize pydantic-ai Agent in analyze(), or use Claude API directly
- Keep imports at top of analyze() or module level for lazy loading

**New Utilities for Frame Processing:**
- Shared helpers: `agent/src/agent/utils.py` or similar
- Export pattern: Import in analyze() or call directly from __main__.py
- Example: Pre-processing filters (brightness normalization, edge detection) before LLM submission

**Test Files (when adding tests):**
- Co-located pattern: `agent/src/agent/prompt_test.py` alongside module
- Or separate: `agent/tests/test_prompt.py` (choose one pattern consistently)
- Use pytest or unittest (not yet specified in requirements)
